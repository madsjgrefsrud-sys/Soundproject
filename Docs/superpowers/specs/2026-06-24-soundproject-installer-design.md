# Soundproject One-Click Installer — Design

Date: 2026-06-24
Status: Approved

## 1. Summary

Soundproject currently ships as a PyInstaller `--onedir` build (`Python/dist/Soundproject/Soundproject.exe`) — a folder you have to copy somewhere and run, with the ViGEmBus driver installed as a separate manual step from a third-party download page. The project is also moving toward a wider public release, as open source.

This round wraps that existing build into a single `Soundproject-Setup-<version>.exe` (built with Inno Setup) that:
- Installs the app like any normal Windows program (Program Files, Start Menu shortcut, optional Desktop shortcut, proper uninstaller in "Apps & Features").
- Silently installs the ViGEmBus driver as part of the same install, using the MSI that the `vgamepad` Python package already bundles — no separate download, no separate manual step.
- Shows a GPLv3 License Agreement page, since the project is being open-sourced under GPLv3.

Nothing about the app's own code changes. This is packaging-only.

## 2. Why GPLv3

The app depends on PyQt6, which Riverbank Computing dual-licenses: free use requires the GPL, commercial use requires a paid Riverbank license. Since this project uses the free PyQt6, distributing the app at all already obligates a GPL-compatible license — GPLv3 isn't just a stylistic choice here, it's the license the PyQt6 dependency requires. The rest of the dependency tree (mido, python-rtmidi, pycaw, pywin32, comtypes, vgamepad) is MIT/permissive, so nothing else conflicts.

## 3. Build flow

```
Python/
  packaging/
    soundproject.spec     (existing, unchanged)
    icon.ico               (existing, unchanged)
    installer.iss           NEW — Inno Setup script
  app/, main.py, ...        (unchanged)
LICENSE                     NEW — repo root, full GPLv3 text
```

Build order (replaces the last step of the README's existing "Build the standalone .exe" section):

1. `python packaging/make_icon.py` (unchanged)
2. `pyinstaller packaging/soundproject.spec` → produces `dist/Soundproject/` (unchanged)
3. `iscc packaging/installer.iss` → produces `dist/Soundproject-Setup-<version>.exe` (new)

Step 3 needs the free [Inno Setup compiler](https://jrsoftware.org/isinfo.php) (`ISCC.exe`) on the machine doing the build. This is a one-time dev-machine setup, same category as already needing PyInstaller — end users never install or see Inno Setup itself, only the resulting `Setup.exe`.

Target is x64 only, matching the existing build (confirmed bootloader is `Windows-64bit-intel`). The installer uses `ArchitecturesAllowed=x64compatible` and only ever runs the x64 ViGEmBus MSI; the x86 MSI that ships inside `vgamepad` is simply not used.

## 4. Installer behavior

Wizard flow: Welcome → License Agreement (GPLv3, must accept to continue) → install location (defaults to `{autopf}`, i.e. `Program Files`) → optional "Create a desktop icon" checkbox (Start Menu shortcut always created) → Install → Finish.

`PrivilegesRequired=admin` — the whole installer runs elevated once. This is required anyway because installing a driver requires admin rights, and it means the user sees **exactly one** UAC prompt for the entire install, not one for the app and a separate one later for the driver.

Driver install is a post-file-copy `[Run]` entry:

```
msiexec /i "{app}\_internal\vgamepad\win\vigem\install\x64\ViGEmBusSetup_x64.msi" /quiet /norestart
```

This runs with the installer's inherited elevation (no second prompt) and references the MSI already sitting in the app's own files — `soundproject.spec` already copies all of `vgamepad/win` into the build (`datas=[(VGAMEPAD_DIR/"win", "vgamepad/win")]`), so no new packaging step is needed to get the MSI there. If ViGEmBus is already installed (e.g. by Discord, Steam, an emulator) `msiexec` no-ops almost instantly — no version-check scripting needed.

Uninstall (auto-generated by Inno Setup) removes only Soundproject's own files and registry entries. ViGEmBus is deliberately left installed, since it's a shared system driver other software may also depend on.

## 5. Licensing

- New `LICENSE` file at the repo root: the canonical GPLv3 text, fetched verbatim from `https://www.gnu.org/licenses/gpl-3.0.txt` (not reconstructed from memory, to guarantee it's exact).
- `installer.iss` sets `LicenseFile` to point at it, producing the License Agreement wizard page described above.
- `Python/README.md` gets a one-line "License: GPLv3" note.

## 6. Versioning

The project has no version number today. This introduces a single one: `1.0.0`, defined as `#define MyAppVersion "1.0.0"` at the top of `installer.iss`, used for both `AppVersion` and the output filename (`Soundproject-Setup-1.0.0.exe`).

There's no build automation in this project (no CI, no git remote configured at all yet), so this is a plain manual constant — bump it by hand in `installer.iss` next release. Not wired into the GUI or anywhere else in the Python code; the app itself has no notion of its own version today, and adding one isn't needed for this installer to work.

## 7. Verification plan

- Build the installer locally and run it on this machine: confirm the License page shows real GPLv3 text and blocks Next until accepted, confirm install completes with exactly one UAC prompt, confirm Start Menu shortcut launches the app.
- Confirm ViGEmBus ends up installed/running via `Get-Service ViGEmBus` after a clean run on a machine where it wasn't already present — can't fully verify this on this dev machine since ViGEmBus is now already installed here from the earlier manual install; will check the `[Run]` step executes (msiexec exit code) instead, and reason about correctness from the command rather than a true clean-machine test.
- Uninstall via "Apps & Features", confirm the app's files are gone and `Get-Service ViGEmBus` still shows it present.

## 8. Explicitly out of scope for this round

- Code signing the `Setup.exe` (costs money; unsigned + a README note about the SmartScreen warning is the agreed trade-off for a free public hobby project).
- An auto-update mechanism. Each release is a manually-built, manually-distributed `Setup.exe`.
- Hosting/distribution (e.g. setting up a GitHub repo and Releases page) — this project has no git remote configured at all yet; that's a separate task from building the installer itself.
- A third-party-licenses/credits dialog in the app for the MIT-licensed dependencies. GPLv3 only requires preserving their existing copyright notices (already intact in the installed packages), not a runtime credits screen.
- Surfacing the version number anywhere in the app's own UI (e.g. an About box).
- Removing ViGEmBus on uninstall, or any 32-bit (`x86`) support path.
